1. Introduction

1.1 Purpose

This document outlines the software requirements for the IFI Farmer Data & Programme Management Platform, a custom application developed by Vivax for the International Foundation for Innovations (IFI). It aims to serve as a clear blueprint for the development, testing, and deployment of the platform.

1.2 Scope

This document defines the requirements for a digital platform to centralize farmer data, track field programmes, disseminate information, and manage appointments. The solution is divided into three primary components:

Field Agent Mobile App: For on-the-ground data collection (Farmer Database, Programme Logging).

Farmer Mobile App (or Web Portal): For information access and booking (Govt. Schemes, Appointments).

Admin Dashboard: For complete platform control, data management, and monitoring.

1.3 Audience

This document is intended for:

Vivax development and QA teams

Project stakeholders at IFI

Client-side decision-makers

DevOps and support teams

1.4 Definitions & Abbreviations

IFI: International Foundation for Innovations

Farmer: The end-beneficiary, user of the Farmer App.

Field Agent: IFI staff member using the Agent App for data collection.

Admin: IFI management staff overseeing the platform via the Admin Dashboard.

CRUD: Create, Read, Update, Delete.

RBAC: Role-Based Access Control.





2. Project Overview

2.1 Business Requirements

To create a comprehensive and centralized database of farmers, including personal, land, and livestock details.

To digitize the logging and tracking of all ongoing field programmes, including participants, media, and outcomes.

To provide farmers with an accessible, up-to-date repository of government schemes.

To streamline the process for farmers to contact IFI and book appointments.

To enable data-driven reporting and monitoring for IFI management.

3. System Features & Requirements

3.1 Farmer App (The "Tanak Prabha" App)

Based on the provided designs, this app is for farmer self-service.

Language Selection & Onboarding:

User must choose a language (e.g., English, Hindi) upon first launch.

A welcome screen introduces the app.

Authentication:

Simple OTP-based login/registration using a mobile number.

Module 1: Farmer Self-Onboarding & Profile Management:

During onboarding, the farmer must enter their own data.

Personal & Family Details: Input fields for Name, Photo, Father's Name, Aadhaar, Mobile Number, Family Details, etc.

Land Holdings & Crop Details: A form to enter total land area (in Bighas) and the crops they grow in different seasons (Rabi, Kharif, Jayad).

Livestock Details: A form to enter the numbers of animals they own (Cow, Buffalo, etc.).

My Information: A profile section where farmers can view and edit all the data they have entered.

Home Dashboard:

A main screen showing weather , "Quick Actions" , "Latest Updates" , and "Programme Happening Today".

Module 3: Information - Government Schemes:

A searchable and filterable list of government schemes.

Farmers can click a scheme to read details, eligibility, and benefits.

Module 4: Contact & Book Appointment:

A dedicated section to book an appointment with IFI staff.

A "Contact" or "Support" page with IFI contact details.

Notifications:

Receive push notifications for appointment confirmations and new scheme alerts.

3.2 Field Agent App

This app is for IFI staff in the field. Offline capability is its most critical feature.

Authentication:

Secure login with a username and password (credentials created by Admin).

Offline-First Design:

All forms (especially Module 2) must be fillable and save locally without an internet connection.

Data will auto-sync with the server when connectivity is restored.

Module 2: Ongoing Programmes (Primary Role):

This is the agent's main job.

A form to create a new "Ongoing Programme" entry.

Input fields for Programme Name, Subject, Trainers, Date, Location, etc.

Participant List: Ability to search and add participants by searching the existing Farmer Database (who registered via the Farmer App).

Media Upload: Use the phone's camera/gallery to upload photos (5-10) and videos (1-2 mins) for the event.

Outcome: A rich-text field to enter the programme's outcome.

Module 1: Farmer Assistance & Verification (Secondary Role):

View Farmer Database: Search, filter, and view all farmers who have registered.

Farmer Assistance: A dedicated flow for the agent to register a farmer who cannot do it themselves (e.g., no smartphone). This form will contain all the Module 1 fields (Personal, Land, Livestock).

Data Verification: (Optional) An agent may have the permission to "verify" the data submitted by a farmer, marking it as "Approved" in the system.

3.3 Admin Dashboard (All Modules)

Secure Login: Admin username and password.

Analytics Dashboard: Overview widgets (e.g., Total Farmers, Programmes This Month, Pending Appointments).

Module 1: Farmer Database Management:

Full CRUD operations on all farmer records.

Advanced search and filtering (by Village, Block, District, etc.).

View farmer geo-locations on an interactive map.

Data Export: Ability to export farmer data (filtered or full list) to CSV/Excel.

Module 2: Programme Management:

Full CRUD operations on all programme entries.

View all details, including participant lists.

View uploaded photos and videos in a gallery.

Search/filter programmes by location, trainer, or date.

Module 3: Government Scheme Management (CMS):

A Content Management System (CMS) to Add, Edit, and Delete government schemes that appear in the Farmer App.

Module 4: Appointment Management:

View all incoming appointment requests.

Update appointment status (Confirm, Cancel, Mark as Completed).

Assign appointments to specific staff/field agents (optional).

User Management:

Create, Edit, and Deactivate Field Agent accounts.

View a list of all registered Farmers.

5. System Architecture

5.1 Monolithic Architecture

Overview:

For the initial version, all modules—farmer management, programme logging, scheme info, and appointments—will reside in a single backend application (e.g., ExpressJS/Node.js). The mobile apps and the admin dashboard will all interact with this single backend and a shared database.

Components:

React Native Farmer App

React Native Field Agent App

Admin Dashboard (React)

Single ExpressJS Backend

Shared Database (PostgreSQL or MongoDB)

SMS Gateway (e.g., Twilio, MSG91) for OTP

File Storage (e.g., AWS S3, BunnyCDN) for photos/videos

Advantages:

Rapid development and deployment.

Lower initial infrastructure complexity.

Easier for a small team to manage.

Limitations:

Scaling is all-or-nothing (the entire app must be scaled).

A failure in one module can potentially affect the entire system.
